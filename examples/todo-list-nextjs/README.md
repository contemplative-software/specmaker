# Todo List Next.js Example

This example demonstrates SpecMaker's core workflows: project initialization and document review.

## What's Inside

This example contains:

- **implementation-plan.md** - A Next.js todo list app implementation plan serving as the input manuscript
- **.specmaker/** - Generated project structure from `init_verify.py`
  - `project_context.json` - Project metadata and configuration
  - `manifest.json` - Manifest of generated files
  - `README.md` - Human-readable project description
- **review-output.json** - Sample review report generated by `review_validation.py`

## What This Demonstrates

### 1. Project Initialization (`init`)

The `init` workflow creates a `.specmaker/` directory with:
- Project context (name, description, audience, style rules)
- File manifest tracking generated artifacts
- Human-readable README

### 2. Document Review (`review`)

The `review` workflow analyzes manuscripts and produces:
- Overall status (pass/changes_required/blocked)
- Detailed summary of the document quality
- Specific issues categorized by type (clarity, accuracy, structure, grammar, style)
- Confidence score for the review

## How This Was Generated

### Step 1: Create Implementation Plan
```bash
# Created implementation-plan.md with a todo list app specification
```

### Step 2: Initialize Project
```bash
cd specmaker-core
uv run --frozen scripts/init_verify.py \
  --root-dir ../../examples/todo-list-nextjs \
  --project-name "Todo List Next.js" \
  --description "A simple todo list application built with Next.js to demonstrate basic CRUD operations"
```

### Step 3: Generate Review (Sample)
```bash
# Note: Actual review requires OpenAI API key
# The included review-output.json is a sample showing expected output format
cd specmaker-core
uv run --frozen scripts/review_validation.py \
  --path ../../examples/todo-list-nextjs/implementation-plan.md \
  --title "Todo List Next.js Implementation Plan"
```

## Learning from This Example

Examine each file to understand:

1. **Input Format** - How to structure implementation plans and manuscripts
2. **Project Context** - What metadata SpecMaker tracks for each project
3. **Review Output** - How SpecMaker categorizes and reports document issues

## Next Steps

To use SpecMaker with your own documents:

1. Create an implementation plan or specification document
2. Run `init` to set up your project structure
3. Run `review` to get feedback on your documents (requires API key)
4. Iterate based on review feedback

For more examples and documentation, see the main SpecMaker repository.
